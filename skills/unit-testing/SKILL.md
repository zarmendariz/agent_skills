# Embedded C Unit Testing

Unit testing embedded C using Unity + CMock + CException, orchestrated by Ceedling (v1.0.0).

## Testing Stack

| Component | Role |
|-----------|------|
| **Unity** | Lightweight C test framework (3 files: unity.c, unity.h, unity_internals.h) |
| **CMock** | Auto-generates mock functions from C header files (Ruby generator + C runtime) |
| **CException** | setjmp/longjmp-based exception handling for C (2 files) |
| **Ceedling** | Ruby/Rake build automation orchestrating all three |

## Project Structure

```
project/
├── project.yml              # Ceedling configuration
├── src/
│   ├── module.c
│   └── module.h
├── test/

│   └── support/             # Shared test helpers
└── build/                   # Generated (gitignore)
    ├── test/out/            # Per-test-executable subdirs (1.0.0)
    ├── test/mocks/          # Generated CMock files
    └── test/runners/        # Generated test runners
```

## Creating Test Modules

Generate with Ceedling: `ceedling 'module:create[module_name]'`

Template:

```c
#include "unity.h"
#include "module_name.h"
#include "mock_dependency.h"

void setUp(void)    { /* init fixtures, reset state */ }
void tearDown(void) { /* cleanup, verify mocks */ }
```

## Writing Tests

### Naming Convention

```
test_<function>_<scenario>_<expected_result>
```

### Structure (Arrange-Act-Assert)

```c
void test_function_returns_expected(void)
{
    int input = 5;                          // Arrange
    int result = function_under_test(input); // Act
    TEST_ASSERT_EQUAL(10, result);          // Assert
}
```

### Key Assertions

| Assertion | Purpose |
|-----------|---------|
| `TEST_ASSERT_EQUAL(exp, act)` | Integer equality |
| `TEST_ASSERT_EQUAL_INT32(exp, act)` | Sized integer |
| `TEST_ASSERT_TRUE/FALSE(cond)` | Boolean check |
| `TEST_ASSERT_NULL/NOT_NULL(ptr)` | Pointer check |
| `TEST_ASSERT_EQUAL_STRING(exp, act)` | String comparison |
| `TEST_ASSERT_EQUAL_HEX8/16/32(exp, act)` | Hex display |
| `TEST_ASSERT_EQUAL_MEMORY(exp, act, len)` | Raw bytes |
| `TEST_ASSERT_FLOAT_WITHIN(delta, exp, act)` | Float tolerance |
| `TEST_ASSERT_EQUAL_INT_ARRAY(exp, act, n)` | Array compare |

Full API: See [references/api_reference.md](references/api_reference.md)

## CMock Essentials

Mock external dependencies by including `mock_header.h`:

```c
#include "mock_hardware_driver.h"  // auto-generated by Ceedling
```

### Mock Functions (Always Available)

| Pattern | Usage |
|---------|-------|
| `func_Expect(args...)` | Expect call, verify args |
| `func_ExpectAndReturn(args..., val)` | Expect call, return value |

### Mock Functions (Plugin-Dependent)

| Pattern | Plugin | Usage |
|---------|--------|-------|
| `func_ExpectAndThrow(args..., err)` | `:cexception` | Throw exception |
| `func_Ignore()` | `:ignore` | Ignore all calls |
| `func_IgnoreAndReturn(val)` | `:ignore` | Ignore, return value |
| `func_IgnoreArg_<param>()` | `:ignore_arg` | Skip one argument |
| `func_ReturnThruPtr_<param>(val)` | `:return_thru_ptr` | Write to output pointer |
| `func_Stub(callback)` | `:callback` | Replace with callback |
| `func_AddCallback(callback)` | `:callback` | Callback after arg check |
| `func_ExpectAnyArgs()` | `:expect_any_args` | Any arguments |

For advanced patterns: See [references/cmock-patterns.md](references/cmock-patterns.md)

## CException

Enable in `project.yml`: `:project: :use_exceptions: TRUE`

```c
void test_throws_on_invalid_input(void)
{
    CEXCEPTION_T e;
    Try { module_process(NULL); TEST_FAIL_MESSAGE("No exception"); }
    Catch(e) { TEST_ASSERT_EQUAL(ERROR_INVALID_INPUT, e); }
}
```

Limitations: no `return`/`goto` inside Try; mark modified locals `volatile`.

## Running Tests

```bash
ceedling test:all                              # All tests
ceedling test:module_name                      # Specific module
ceedling "test:pattern[Init]"                  # Pattern match
ceedling test:foo --test-case=configure        # Filter cases
ceedling gcov:all                              # With coverage
ceedling clobber test:all                      # Clean + test
ceedling --mixin=mixins/clang.yml test:all     # With Clang toolchain
```

## Compiler Configuration

### Switching GCC → Clang (Minimal)

```yaml
:tools_test_compiler:
  :executable: clang
:tools_test_linker:
  :executable: clang
```

### With Sanitizers

```yaml
:flags:
  :test:
    :compile:
      :*:
        - -fsanitize=address,undefined
        - -fno-omit-frame-pointer
        - -O1 -g
    :link:
      :*:
        - -fsanitize=address,undefined
```

## Best Practices

1. **Isolate tests** — independent, no shared state, mock all externals
2. **Use HAL** — abstract hardware behind mockable interfaces
3. **Inject dependencies** — pass values as parameters, not direct HW reads
4. **Cover >80%** — happy paths, error cases, edge cases
5. **Manage fixtures** — `setUp()` inits, `tearDown()` verifies and cleans
6. **Use sanitizers** — ASan + UBSan catch memory/UB bugs in test builds
7. **Parallel builds** — set `:compile_threads: :auto` in project.yml
