# Embedded C Unit Testing

Unit testing embedded C using Unity + CMock + CException, orchestrated by Ceedling.

## Testing Stack

- **Unity**: Lightweight C test framework for embedded
- **CMock**: Auto-generates mocks from C headers
- **CException**: C exception handling (optional)
- **Ceedling**: Build automation orchestrating all three

## Project Structure

```
project/
├── test/test_module_name.c
├── src/module_name.{c,h}
└── project.yml
```

## Creating Test Modules

Generate with Ceedling: `ceedling module:create[module_name]`

Template:

```c
#include "unity.h"
#include "module_name.h"
#include "mock_dependency.h"

void setUp(void)    { /* init fixtures, reset state */ }
void tearDown(void) { /* cleanup, verify mocks */ }
```

## Writing Tests

### Naming Convention

```
test_<function>_<scenario>_<expected_result>
```

### Structure (Arrange-Act-Assert)

```c
void test_function_returns_expected(void)
{
    int input = 5;                          // Arrange
    int result = function_under_test(input); // Act
    TEST_ASSERT_EQUAL(10, result);          // Assert
}
```

### Key Assertions

| Assertion | Purpose |
|-----------|---------|
| `TEST_ASSERT_EQUAL(exp, act)` | Integer equality |
| `TEST_ASSERT_TRUE/FALSE(cond)` | Boolean check |
| `TEST_ASSERT_NULL/NOT_NULL(ptr)` | Pointer check |
| `TEST_ASSERT_EQUAL_STRING(exp, act)` | String comparison |
| `TEST_ASSERT_EQUAL_HEX8/16/32(exp, act)` | Hex display |
| `TEST_ASSERT_FLOAT_WITHIN(delta, exp, act)` | Float tolerance |

## CMock Essentials

Mock external dependencies (GPIO, UART, SPI, I2C, other modules) by including `mock_header.h`:

```c
#include "mock_hardware_driver.h"  // auto-generated by Ceedling
```

### Mock Patterns

| Pattern | Usage |
|---------|-------|
| `func_Expect()` | Expect call, verify args |
| `func_ExpectAndReturn(val)` | Expect call, return value |
| `func_ExpectAndThrow(err)` | Expect call, throw exception |
| `func_StubWithCallback(cb)` | Custom behavior |

For advanced patterns and strict ordering: See [references/cmock-patterns.md](references/cmock-patterns.md)

## CException

Enable in `project.yml`: `:cexception: :enabled: true`

```c
void test_throws_on_invalid_input(void)
{
    CEXCEPTION_T e;
    Try { module_process(NULL); TEST_FAIL_MESSAGE("No exception"); }
    Catch(e) { TEST_ASSERT_EQUAL(ERROR_INVALID_INPUT, e); }
}
```

## Running Tests

```bash
ceedling test:all                              # all tests
ceedling test:module_name                      # specific module
ceedling clobber test:all gcov:all utils:gcov  # with coverage
```

## Best Practices

1. **Isolate tests** — independent, no shared state, mock all externals
2. **Use HAL** — abstract hardware behind mockable interfaces
3. **Inject dependencies** — pass values as parameters, not direct HW reads
4. **Cover >80%** — happy paths, error cases, edge cases
5. **Manage fixtures** — `setUp()` inits, `tearDown()` verifies and cleans
